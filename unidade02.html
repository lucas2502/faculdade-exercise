#Managed Bean

Managed Beans são classes Java Beanque servem de comunicação entre uma página Web e asclasses de persistência da sua aplicação. É com elas que nós recebemos os valores submetidospelos usuários, chamamos métodos de ouvintes para responder a ações do usuário efornecemos dados para serem exibidos em uma página.A grande vantagem de utilizarmos managed beans é que eles não permitem desenvolver ainteligência da aplicação dentro da visão, ou seja, implementar as regras de negócio na própriapágina Web. Tal prática é abolida justamente por infringir o padrão MVC.

@NoneScoped: o managed bean será instanciado a cada vez que for referenciado.

@RequestScoped: o managed bean será instanciado uma única vez dentro da requisição HTTP edescartado assim que a resposta for enviada ao usuário.

@ViewScoped: o managed bean será instanciado e mantido ativo até que o usuário navegue para umapróxima página.

@SessionScoped: o managed bean será instanciado e mantido ativo até que a sessão do usuário sejainvalidada ou o tempo limite seja atingido. Como cada usuário possui sua própria sessão, o managedbean não pode ser compartilhado entre os usuários.

@ApplicationScoped: o manged bean será instanciado e mantido ativo durante todo o tempo deexecução da aplicação. Permite que um maneged bean seja compartilhado com todos os usuários.

#O Ciclo de Vida do JSF

Para desenvolver uma aplicação Web em JSF, é muito importante conhecer seu ciclo de vida.Ao executarmos uma página construída com essa API, as seguintes fases serão realizadas:

1. Restauração da visão.

2. Aplicação de valores da requisição.

3. Processamento de validações.

4. Atualização de valores do modelo.

5. Invocação da aplicação.

6. Renderização da resposta.

#JavaServer Faces (JSF)

JavaServer Faces (JSF) é um framework de interfaces do usuário que foi projetado para facilitara codificação de aplicações Web (NETBEANS, 2015). De fato, JSF é uma especificação da JEEque utiliza um modelo de interfaces do usuário orientado a eventos para desenvolver projetosWeb (FARIA, 2013).Fazendo-se uma analogia com o desenvolvimento de aplicações desktop, podemos dizer queJSF é uma biblioteca gráfica, assim como as bibliotecas Java AWT e Swing, só que paradesenhar interfaces de aplicações Web.A vantagem de utilizar JSF é que essa API torna o desenvolvimento de aplicações Web menoscomplexo e mais produtivo se comparado com as API Java Servlets e JavaServer Pages. Nãoquer dizer que JSF substitui essas bibliotecas. Ela apenas acrescenta recursos que simplificame muito a vida dos desenvolvedores.
